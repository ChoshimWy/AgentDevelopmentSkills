1. 范围与成功标准

Phase 3 目标将“设计体验能力”“设计真源读取”“Canonical IR 编译”“平台 UI Binding”拆为独立 Capability，保证任何实现输入都能回溯 source slice、版本、parser 和人工合同。

本阶段包含

  • Product Design Provider capability 映射。
  • Figma / Sketch / manual / screenshot Adapter 只读合同。
  • Design Evidence、Canonical IR、Registry、Agent Packet Schema。
  • Apple UI Binding 与最小视觉验收证据。

本阶段不包含

  • 不把 Product Design 当作唯一设计基础设施。
  • 不允许截图单独成为完整设计事实。
  • 不默认写回 Figma / Sketch。
  • 不在 Core 内实现 SwiftUI/UIKit 代码生成。

阶段成功标准

  1. source、manual-contract、platform-binding、inference、unknown provenance 不混用。
  2. Agent Packet 仅包含 task-scoped slice，不包含完整设计文档或凭据。
  3. 设计写操作必须通过精确 document/page/node approval gate。
  4. Apple Binding 变化不会改变 Canonical IR 的平台无关语义。
