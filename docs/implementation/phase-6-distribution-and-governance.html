阶段成功标准

  1. 同 Lockfile、Profile 和环境 baseline 生成相同 Capability Binding 与 Plan fingerprint。
  2. 所有生命周期动作先 dry-run，写入有事务边界和恢复记录。
  3. 未知来源、hash 篡改、权限扩大、Schema 不兼容和冲突全部 fail-closed。
  4. 单一发布包在 Python 3.11+ 干净环境分别完成 core-only、单平台、多平台和 all 的 install → doctor → route → uninstall smoke。
  5. 本机 Xcode 官方知识源只以 source/build/SDK/content/routing hash 进入审计，不把导出正文或工具再分发到 release artifact。
