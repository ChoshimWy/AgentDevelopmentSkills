1. 范围与成功标准

2026-07-15 · Source Install Anchor根目录 install.sh 已实现 Manifest 驱动的平台复选：TTY 以 [x] / [ ] 列出 Apple/iOS、Android、Web、Backend、Desktop，使用 ↑ / ↓ 移动光标、Space 选择或取消、Enter 确认，确认后清除整个平台菜单；平台只有同时满足 Manifest implemented 与 source-installer activation/smoke handler readiness 才可选；当前仅 Apple 默认勾选，其他平台以 bootstrap-only 明确展示并阻塞；非交互模式支持可重复 --platform / --platform all。当前安装闭包为 Apple + shared disciplines + 显式 Codex Runtime Config，支持 dry-run 与 target-root，默认人类可读结果直接从所选平台状态开始且不重复显示产品标题/分隔线，并保留变更/验证摘要，完整安装元信息由 --json 以 canonical JSON 输出；可直接迁移精确识别的旧 iOSAgentSkills 软链且不生成持久备份，保留 skills/.system，激活 config/profile/agents/bin/templates，并对实际目标执行 installed workflow smoke；后置 smoke/activation 与受管根共享单进程临时回滚窗口，临时数据完成后删除。dry-run 与真实安装共用 managed-root preflight；普通文件形式的 macOS .DS_Store 作为非执行元数据忽略，其他未知本地内容与 activation hash 漂移继续 fail-closed；完整 unittest、Conformance 与独立 reviewer 复审通过，阻塞问题：无。
2026-07-16 · Apple Official Expertise Federation Anchor已增加本机 Xcode 官方 Skill 导出的只读联邦适配器、packet Schema、审查路由表与 7 个稳定 Capability Binding。适配器仅信任 Xcode 默认受管目录或用户显式核实的非默认导出，不联网、不执行导出脚本、不复制正文；unverified/unknown Skill、Xcode identity 缺失、SDK 不满足、symlink 或冻结内容 hash 漂移均 fail-closed。现有 ios-feature-implementation、xcode-build、ios-automation、apple-debugging 与 apple-code-review 仍是唯一执行入口。当前主机为 Xcode 26.5，因此 Xcode 27 live export smoke 尚未执行，不得宣称 host-ready。
P2C 与 Phase 6 边界当前入口仍是 source-checkout installer；doctor、持久化 lock 生命周期、upgrade/多平台部分卸载、跨进程恢复、wheel/sdist 与 SBOM 仍未完成；2026-07-15 已完成真实 ~/.codex source install，但不等同于发布包完成。
2026-07-16 · Verified Bootstrap Anchor已按“两个薄 bootstrap + 一个共享跨平台安装 Core + 经过校验的发布物”落地首个 distribution anchor：根目录 install.sh 保留 source-checkout fast path；托管模式与新增的 Windows install.ps1 均先读取 canonical release-manifest.json，从版本固定的 asset_base_url 下载并校验共享 bootstrap_install.py 后才执行。共享 Core 按 host 唯一选择 artifact、验证 size/SHA-256，拒绝非 HTTPS、危险重定向、symlink、path traversal、ZIP 规范化别名、大小写或 Unicode normalization 冲突和解压上限越界；build_release_bundle.py 生成 byte-stable ZIP、bootstrap assets、source identity 与 manifest。默认 production host envelope 仅为 darwin/linux（WSL2 复用 Linux）；Windows production manifest 继续 fail-closed。公开 distribution host、stable release、签名/SBOM、wheel/sdist 与完整 release gate 仍未完成，不得宣称发布就绪。
Phase 6 目标将当前“源码树可运行”提升为“可安装、可锁定、可升级、可审计、可回滚、可发布”的产品化工作流，同时保持离线优先与 fail-closed。

本阶段包含

  • agent-skills.lock 生成、校验、diff 与 plan fingerprint。
  • 仓内 registry、--core-only/单平台/多平台/all 安装、doctor、upgrade、uninstall 与 rollback。
  • permission profile、供应链 hash、审批与机密数据规则。
  • Local / CI / Remote compatible evidence 与 release Conformance。

本阶段不包含

  • 不默认建设公网插件市场或远程执行平台。
  • 不自动下载、安装或升级未知来源插件。
  • 不允许安装器覆盖用户本地自定义内容。
  • 不允许 migration/rollback 丢失原版本 hash。

阶段成功标准

  1. 同 Lockfile、Profile 和环境 baseline 生成相同 Capability Binding 与 Plan fingerprint。
  2. 所有生命周期动作先 dry-run，写入有事务边界和恢复记录。
  3. 未知来源、hash 篡改、权限扩大、Schema 不兼容和冲突全部 fail-closed。
  4. 单一发布包在 Python 3.11+ 干净环境分别完成 core-only、单平台、多平台和 all 的 install → doctor → route → uninstall smoke。
  5. 本机 Xcode 官方知识源只以 source/build/SDK/content/routing hash 进入审计，不把导出正文或工具再分发到 release artifact。
