1. 范围与成功标准

Phase 2 目标用版本化 Manifest、Capability Contract 与执行 Adapter 把 Apple 工作流绑定到 Core DAG；相同任务可在旧入口与 Core 入口间对比，并能在不兼容或缺失时 fail-closed。

本阶段包含

  • Apple Capability inventory、Manifest 与兼容矩阵。
  • Core request / Apple entrypoint / structured evidence 的 Adapter。
  • 原入口与 Core 路由的双路径 golden、真实仓 dry-run 与最小 smoke。
  • 缺失包、不兼容版本、权限扩大和 Provider 冲突的负向验证。

本阶段不包含

  • 不把 Apple API、Xcode 命令或设备策略移入 Core。
  • 不删除、重命名或弱化 iOSAgentSkills 现有入口。
  • 不接入 Design Gateway、QA Core 或其他平台真实执行。
  • 不切换私有 Pod 依赖或修改业务项目代码。

阶段成功标准

  1. Core 只通过稳定 Capability ID 解析 Apple Provider。
  2. Apple 包独立使用能力保持不变，Core 接入可通过开关关闭。
  3. verification / review / report 证据可通过现有 v1 Schema 或经评审后的最小扩展表达。
  4. 所有 mandatory 能力缺失、版本不兼容和权限扩大均在执行前阻塞。
