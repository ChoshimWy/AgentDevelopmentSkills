1. 范围与成功标准

Phase 2 目标用版本化 Manifest、Capability Contract 与执行 Adapter 把 Apple 工作流绑定到 Core DAG，并将完整 Apple AgentSkills 迁入同一仓库；相同任务可在旧入口与仓内 Core 入口间对比，安装时可显式选择 Apple，缺失或不兼容时 fail-closed。

本阶段包含

  • Apple Capability inventory、Manifest 与兼容矩阵。
  • Core request / Apple entrypoint / structured evidence 的 Adapter。
  • 原入口与 Core 路由的双路径 golden、真实仓 dry-run 与最小 smoke。
  • 将 iOSAgentSkills 的 Apple 专属资产迁入 platforms/apple/，将跨平台通用资产唯一迁入 disciplines/。
  • Apple 单平台、组合平台与未选择 Apple 的安装态验证。

本阶段不包含

  • 不把 Apple API、Xcode 命令或设备策略移入 Core。
  • 迁移验收前不删除、重命名或弱化 iOSAgentSkills 现有入口。
  • 不接入 Design Gateway、QA Core 或其他平台真实执行。
  • 不切换私有 Pod 依赖或修改业务项目代码。

阶段成功标准

  1. Core 只通过稳定 Capability ID 解析 Apple Provider。
  2. platforms/apple/ 是 Apple 专属资产唯一安装真源，共享资产由 disciplines/ 唯一提供，安装和运行不依赖 sibling iOSAgentSkills 仓。
  3. 安装器能选择 Apple、多个平台、全部平台或仅 Core，未选择 Apple 时不安装其 Skills 和权限。
  4. verification / review / report 证据可通过现有 v1 Schema 或经评审后的最小扩展表达。
  5. 所有 mandatory 能力缺失、版本不兼容和权限扩大均在执行前阻塞。
